Default to an index page  If the user specifies a directory, rather than a filename, in the URI, Interchange shows a "not found" page instead of looking for a default file such as index.html.  After a short thread, Rene Hertell kindly summarised his simple solution.  Interchange versions 4.9.3 and above have an Apache-style DirectoryIndex directive which makes this redundant. This information is still useful to users who have not yet upgraded from an earlier Interchange version.
