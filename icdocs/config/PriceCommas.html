PriceCommas

Format price values using the thousands separator.

Synopsis

PriceCommas  No

This is a Yes or No directive.  The default is Yes.

Scope

This directive is only available for use in the local (catalog.cfg) configuration file.  It will not affect any other website in any way.  This directive will not work in the global (interchange.cfg) configuration file. 

Description

If no commas are desired in price values, returned by tags such as [price] etc., then this directive should be set false.  The default is to use thousands separators.

A value of "10000" would show as "10,000" if thousands separators are required (which they usually are).  The comma, used as the thousands separator in this example, is for illustration only.  The actual separator is defined by the "thousands_sep" value, set for the current Locale.

Note

This is overridden if a price_picture is set for the Locale that's in use on a page.

Example

#
#   set a default
#
PriceCommas   1

#
#   establish locale-specific "price commas" values
#
Locale fr_FR  PriceCommas  0
Locale en_GB  PriceCommas  1

I imagine that the French would prefer to see thousands separators in their price values.  This is just an example, after all.
